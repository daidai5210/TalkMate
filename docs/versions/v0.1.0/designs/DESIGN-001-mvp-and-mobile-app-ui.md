## 1. 方案目标

- 完成 TalkMate MVP 的全栈闭环：账号、场景、对话、AI、语音、总结、历史。
- 将移动端体验从响应式网页调整为手机端 App 式任务流。
- 保持现有 Web 技术栈，不引入 Native App 或小程序。
- UI 调整不新增后端接口、不扩大数据模型。

## 2. 适用范围

- 前端：React 18 + TypeScript + Vite + Tailwind CSS。
- 后端：FastAPI + SQLAlchemy + SQLite。
- AI：DeepSeek OpenAI-compatible API。
- 语音：浏览器 Web Speech API。
- 页面：登录、注册、首页、对话、历史回放、总结。
- 视口：375/390/430/768/1440。

## 3. 用户流程

### 主流程

1. 用户进入登录页或注册页。
2. 注册时输入用户名、密码、验证码；登录后获得 JWT。
3. 进入首页，查看 5 个训练场景和历史练习记录。
4. 点击场景任务，创建对话并进入训练页。
5. 用户通过文字或语音输入英文。
6. 后端保存用户消息，调用 AI 生成场景化回复，并保存 AI 消息。
7. 前端展示 AI 回复并可通过 TTS 播放。
8. 用户完成不少于 10 轮对话后进入总结页。
9. 后端生成并保存纠错反馈、评分、词汇分析、改进建议。
10. 用户从首页历史记录回看对话和总结。

### 异常流程

| 场景 | 触发条件 | 系统表现 | 用户提示 | 后续处理 |
|---|---|---|---|---|
| 登录失败 | 用户名不存在或密码错误 | 登录接口返回业务错误 | 展示明确错误 | 用户修改后重试 |
| 注册失败 | 用户名重复、验证码错误、密码不合法 | 注册接口返回业务错误 | 字段或页面错误提示 | 用户修改后重试 |
| 场景加载失败 | API 异常或网络失败 | 首页展示错误态 | 提供重试按钮 | 用户重试 |
| AI 超时 | DeepSeek 响应超时 | 后端降级或返回错误 | 展示发送失败/重试 | 用户重试或继续文字输入 |
| 语音不可用 | 浏览器不支持或权限拒绝 | 语音按钮展示不可用/失败 | 提示改用文字输入 | 保留文字输入 |
| 总结生成失败 | AI 或接口失败 | 总结页展示错误态 | 提供重试 | 用户重试生成 |
| 历史回放 | 从历史进入对话 | 输入区禁用，只读展示 | 显示只读提示 | 可进入总结 |

## 4. 功能模块

| 模块 | 功能说明 | 用户价值 | 规则说明 | 优先级 |
|---|---|---|---|---|
| auth | 注册、登录、登出、JWT | 建立个人练习记录 | 密码 bcrypt，验证码固定值 | P0 |
| scenario | 5 个场景任务 | 快速开始练习 | 种子数据初始化 | P0 |
| conversation | 创建对话、发送消息、历史消息 | 完成英语训练 | 消息持久化，用户隔离 | P0 |
| ai_service | DeepSeek 回复与总结 | 提供智能陪练和反馈 | 超时降级，key 安全注入 | P0 |
| voice | STT/TTS | 模拟口语练习 | 浏览器 API，文字降级 | P0 |
| summary | 纠错、评分、建议 | 形成学习反馈 | 一次对话一份总结 | P0 |
| history | 练习记录和回放 | 支持复盘 | 首页列表入口 | P1 |
| mobile_app_shell | 手机端 App 壳层和任务流 | 提升移动端体验 | 最大宽度、底部操作、安全区 | P0 |

## 5. 页面与交互要求

| 页面/入口 | 用户动作 | 展示内容 | 状态变化 | 异常提示 |
|---|---|---|---|---|
| `/login` | 登录 | 品牌、账号密码表单、主按钮 | 默认/加载/错误/成功跳转 | 用户名或密码错误 |
| `/register` | 注册 | 账号、密码、验证码、主按钮 | 默认/加载/错误/成功跳转 | 用户名重复、验证码错误 |
| `/` | 选择任务 | 推荐训练、5 个场景、历史记录 | 加载/空态/错误/正常 | 加载失败可重试 |
| `/conversation/:id` | 对话训练 | 顶部任务栏、消息列表、底部输入 | 新对话/发送中/AI 回复中/录音中 | 发送失败、录音失败 |
| `/conversation/history/:conversationId` | 回看历史 | 只读消息、总结入口 | 只读态 | 无输入区 |
| `/conversation/:id/summary` | 查看总结 | 评分、纠错、词汇分析、建议 | 生成中/失败/正常 | 失败可重试 |

## 6. 字段与数据规则

| 字段 | 含义 | 类型 | 必填 | 规则 | 示例 |
|---|---|---|---|---|---|
| username | 用户名 | string | 是 | 3-50 字符 | `student01` |
| password | 密码 | string | 是 | 8-32 字符 | `demo1234` |
| captcha | 注册验证码 | string | 是 | MVP 固定 4 位 | `1234` |
| scenario_id | 场景 ID | number | 是 | 必须存在 | `1` |
| message.text | 用户输入 | string | 是 | 1-5000 字符 | `I want to practice interview.` |
| summary.score | 综合评分 | number | 是 | 0-100 | `82` |
| summary.feedback | 纠错反馈 | JSON/text | 是 | 至少包含纠错说明 | `[...]` |
| summary.suggestions | 改进建议 | JSON/text | 是 | 数组或文本 | `[...]` |

## 7. 权限与状态规则

- 除注册、登录、健康检查外，其余业务接口需要 JWT。
- 前端 401 时清理 token 并跳转登录页。
- 用户只能访问自己的对话、消息和总结。
- 历史回放模式不能发送消息。
- 总结只能基于已存在且有权限访问的对话生成。

## 8. 技术与 UI 约束

- 前端主容器使用手机端最大宽度，桌面端居中展示。
- 关键触控控件高度不低于 44px。
- 底部固定操作区考虑 `safe-area-inset-bottom`。
- 禁止横向滚动和文本遮挡。
- Indigo 作为主品牌色，不使用大面积桌面营销式 hero。
- 不提交 `.env`、密钥、`talkmate.db`、`node_modules`、`venv`、`dist`。

## 9. 不做事项

- 不引入 Native App、小程序或跨端框架。
- 不新增第三方登录、个人中心、课程商城、社区、教师端。
- 不在手机端 UI 调整中新增后端接口或数据库字段。
- 不把未确认的 `frontend/src/app/AppLayout.tsx` 底部导航草稿视为已交付方案。
