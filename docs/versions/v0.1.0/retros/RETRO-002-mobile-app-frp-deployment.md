## 3. 经验沉淀

- **FRP 公网部署**：
  - TCP 模式省略 `remotePort` 让服务端分配随机端口是正确做法
  - 但前端 API 地址必须用公网后端地址，不能用本地回环
  - CORS 白名单必须包含公网前端地址（完整协议+域名+端口）
  - Web Speech API 在 HTTP 环境下浏览器会限制，但代码层不应主动禁用，应让浏览器自行判断

- **GitHub 权限管理**：
  - 密钥应保存在 `.local-secrets/` 目录，不在群聊明文传输
  - 使用 Python 脚本从密钥文件读取 token 调用 GitHub API 是安全的方式
  - SSH key 配置后推送非常顺畅

- **开发流程**：
  - 历史 PR 合并后没有删除远程分支，导致 GitHub 分支列表混乱
  - 应按规范：PR 合并后立即删除对应 feature 分支
  - 分支命名应统一格式，当前 `feat/` 和 `feature/` 混用
