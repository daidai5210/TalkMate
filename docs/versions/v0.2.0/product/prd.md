## 2. 背景与目标

### 2.1 背景

- 业务背景：TalkMate 当前 v0.1.0 已支持场景对话练习和 AI 总结反馈，但用户缺少对自己弱点的持续追踪。每次总结反馈后没有沉淀，用户不知道自己的"中式英语"问题到底集中在哪。
- 用户痛点：用户练完后只知道"这次"犯了多少错，但不知道"我是不是老在同一个坑里"。缺乏累积视角和自我纠错方向，练习变成无目的轮回。
- 当前替代方案：用户只能靠记忆或手动回顾历次总结来感知自己的弱项，效率极低。
- 不做的影响：用户留存和复练率缺乏抓手，产品停留在"单次练习工具"而非"持续进步教练"。

### 2.2 产品目标

- 一句话目标：让用户看到自己"中式英语"的具体问题分布，并基于画像给出下一次练习的明确方向。
- 成功标准：
  1. 用户完成一次总结后，能在总结页看到本次错误分布
  2. 用户完成 ≥5 次练习后，首页能看到个性化练习推荐
  3. AI 错误分类准确率 ≥ 70%（以人工抽检 20 条为基准）
- 核心指标：总结页画像卡片曝光率、首页推荐入口点击率、5 次以上练习用户占比

## 4. 业务流程

### 4.1 主流程

```
完成对话 → AI 总结（含 error_type 标签）→ 总结页展示本次错误画像卡片
                                              │
                                              ▼
                                   后台更新 user_error_profiles 表
                                              │
                                              ▼
                     用户下次打开首页 → 看到练习推荐 banner（≥5 次练习后）
                                              │
                                              ▼
                             用户点击推荐 → 进入对应场景开始练习
```

### 4.2 异常流程

| 场景 | 触发条件 | 系统表现 | 用户可继续动作 |
|---|---|---|---|
| AI 未输出有效 error_type | AI 返回格式异常或全部为 unknown | 本次不更新画像，总结页不展示画像卡片 | 正常查看总结 |
| 画像数据为空 | 用户练习次数 < 5 | 总结页不展示画像卡片，首页展示引导文案 | 继续练习 |
| 推荐目标场景不存在 | 对应场景被管理员删除 | 降级推荐最高频错误对应的默认场景 | 进入推荐场景练习 |

### 4.3 空状态

| 页面/模块 | 空状态原因 | 展示内容 | 引导动作 |
|---|---|---|---|
| 总结页-画像卡片 | 本次练习无有效错误分类 | 不展示画像卡片（简洁留白） | 继续查看总结 |
| 首页-练习推荐 | 练习次数 < 5 | "完成 5 次练习后解锁你的中式英语画像" | 引导去练习 |
| 首页-练习推荐 | 画像数据全为 0（用户无典型错误） | "目前没有明显的中式英语倾向，继续加油" | 引导自由练习 |

## 5. 功能需求

| 编号 | 功能 | 优先级 | 用户价值 | 业务规则 | 验收标准编号 |
|---|---|---|---|---|---|
| F-001 | AI 总结增加 5 类错误标签输出 | P0 | 让画像有数据来源 | AI 为每条纠错标注 error_type（word_order/tense/article/preposition/direct_translation/unknown），unknown 不参与聚合 | AC-001, AC-002 |
| F-002 | 后端存储用户错误画像 | P0 | 画像数据持久化 | 新增 user_error_profiles 表，每次总结后增量更新累计计数和滑动窗口（最近 5 次） | AC-003 |
| F-003 | 总结页本次错误画像卡片 | P0 | 让用户在总结时直接看到本次弱项 | 总结页下方展示本次 5 类错误柱状分布，练习 < 5 次时不展示 | AC-004 |
| F-004 | 首页练习推荐 banner | P0 | 实现"发现弱点→定向练习"闭环 | 基于最近 5 次聚合的最高频错误类型推荐对应场景，每次 1 个目标 | AC-005, AC-006 |
| F-005 | 对话删除时画像数据同步 | P1 | 数据一致性 | 删除对话时从滑动窗口移除该次记录，累计计数保留 | AC-007 |

## 6. 非功能约束

| 类型 | 要求 | 是否必需 |
|---|---|---|
| 性能 | 画像聚合查询 < 200ms | 是 |
| 安全 | 用户只能查看自己的画像数据，JWT 认证 | 是 |
| 权限 | 所有新端点需 JWT 认证，按 user_id 隔离 | 是 |
| 兼容性 | 不改变现有对话核心流程和 API 契约 | 是 |
| 可观测性 | 新增 API 端点接入现有日志 | 否（v0.3.0 补充） |
| 部署 | 沿用 Vercel + TiDB，需 1 次 Alembic migration | 是 |

## 9. Q1-Q6 PM 决策

| 编号 | 问题 | PM 决策 |
|---|---|---|
| Q1 | 首 5 次练习前画像为空，首页入口是否显示？ | 显示引导文案"完成 5 次练习后解锁你的中式英语画像" |
