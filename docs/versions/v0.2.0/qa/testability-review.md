## 2. 验收标准可测试性

| 验收标准 | 是否可测试 | 测试方式 | 需要补充 |
|---|---|---|---|
| AC-001：AI 总结返回 JSON 中包含 error_profile 字段，5 个 error_type 各计数值为非负整数 | 是 | API 测试 + 日志 | 需确认测试用的对话内容能触发至少 2 类错误 |
| AC-002：unknown 类型的错误不出现在画像卡片中 | 是 | API 测试 + UI 验证 | — |
| AC-003：user_error_profiles 表每次总结后正确更新累计计数和滑动窗口 | 是 | API 测试 + 数据库查询 | 需准备至少 6 次练习数据验证滑动窗口 |
| AC-004：总结页在练习次数 ≥5 时展示本次错误分布卡片，<5 时不展示 | 是 | UI 测试 | 需 2 个测试账号（1 个 <5 次，1 个 ≥5 次） |
| AC-005：首页 banner 在练习次数 ≥5 时显示推荐场景和重点改善错误类型 | 是 | UI 测试 | 同 AC-004 |
| AC-006：首页 banner 在练习次数 <5 时显示引导文案 | 是 | UI 测试 | 同 AC-004 |
| AC-007：删除对话后滑动窗口正确移除该次记录，累计计数不变 | 是 | API 测试 + 数据库查询 | — |

**结论：** 7 条验收标准全部可测试，无需补充。

---

## 3. 测试范围

| 范围 | 内容 | 是否必测 |
|---|---|---|
| 主流程 | 完成对话 → 生成总结 → 画像更新 → 首页推荐 | 是 |
| 异常流程 | AI 未返回 error_profile / 全 unknown / 空对话 | 是 |
| 权限 | 用户只能查看自己的画像数据，跨用户隔离 | 是 |
| 兼容性 | 旧总结（无 error_profile）不受影响 | 是 |
| 性能 | 画像聚合查询 < 200ms | 否（用户量小，不做专项性能测试） |
| 安全 | JWT 认证，user_id 隔离 | 是 |
| 回归 | 现有对话/总结/场景功能不受影响 | 是 |

## 4. 测试前置条件

| 条件 | 要求 | Owner | 是否具备 |

## 5. 测试用例概要

### 5.1 主流程测试

| 编号 | 用例 | 预期结果 | 覆盖 AC |
|---|---|---|---|
| TC-01 | 用户完成 1 次对话并生成总结 | 总结页展示本次错误画像卡片，user_error_profiles 表有数据 | AC-001, AC-003 |
| TC-02 | 用户完成 5 次对话后打开首页 | 首页展示练习推荐 banner，含推荐场景和错误类型 | AC-005 |
| TC-03 | 用户完成第 6 次对话 | 滑动窗口仅保留最近 5 次，recent_count 正确 | AC-003 |

### 5.2 空状态测试

| 编号 | 用例 | 预期结果 | 覆盖 AC |
|---|---|---|---|
| TC-04 | 新用户（0 次练习）打开首页 | 首页展示引导文案"完成 5 次练习后解锁" | AC-006 |
| TC-05 | 用户完成 3 次练习后打开总结页 | 总结页不展示画像卡片 | AC-004 |

### 5.3 异常流程测试

| 编号 | 用例 | 预期结果 | 覆盖 AC |
|---|---|---|---|
| TC-06 | AI 返回的 error_profile 全为 0 | 总结页不展示画像卡片，不影响总结生成 | AC-002 |
| TC-07 | AI 返回的 error_profile 含 unknown 类型 | unknown 不写入 user_error_profiles，前端不展示 | AC-002 |

### 5.4 权限测试

| 编号 | 用例 | 预期结果 | 覆盖 AC |
|---|---|---|---|
| TC-08 | 用户 A 请求用户 B 的画像数据 | 返回空或 403 | 安全 |
| TC-09 | 未登录请求 /profile/error-summary | 返回 401 | 安全 |

### 5.5 回归测试

| 编号 | 用例 | 预期结果 | 覆盖 AC |
|---|---|---|---|
| TC-10 | 现有对话流程（创建→发消息→结束） | 不受影响 | 回归 |
| TC-11 | 现有总结生成和查看 | 不受影响，旧总结无 error_profile 字段正常展示 | 兼容性 |
| TC-12 | 删除对话后画像滑动窗口同步 | recent_count 重算正确，total_count 不变 | AC-007 |

## 6. 风险与建议

| 风险 | 影响 | 建议处理 | 是否阻断开发 |
|---|---|---|---|
| AI 返回的 error_profile 格式不稳定 | 画像数据不准确 | 后端做字段校验和兜底，测试时用至少 3 种不同对话内容验证 | 否 |
| 测试数据准备耗时 | 测试周期拉长 | 开发阶段提供 seed 脚本快速生成 6 次练习数据 | 否 |
| SQLite 与 TiDB 行为差异 | 生产环境 JSON 字段行为可能不同 | 在 Vercel preview 环境做 1 轮冒烟 | 否 |
