## 1. 背景

- 用户原始诉求：开发 TalkMate AI 英语口语陪练 MVP，并在后续指出当前登录页和页面布局更像网页端，不像原生手机 App，需要调整。
- 当前问题：MVP 功能已形成闭环，但原 S1 需求材料散落在 `docs/product/`，手机端 App 化调整需求散落在聊天进度与 UIUX 讨论稿中，接手开发人员难以从标准目录定位。
- 业务目标：让用户能在浏览器中完成场景化英语口语训练，并获得详细纠错与课后总结；界面体验需从“移动端可用网页”调整为“手机端优先的 AI 口语训练 App”。

## 2. 目标用户

| 用户 | 使用场景 | 目标 |
|---|---|---|
| 大学生 | 英文面试练习 | 提升面试英语表达能力 |
| 职场人士 | 英文会议、面试前练习 | 降低英文表达压力 |
| 出国旅游人群 | 机场、酒店、问路、购物等旅行场景 | 出行前口语突击 |

## 3. 需求合理性评估

- 评估结论：合理可执行。
- 主要依据：
  - MVP 范围已在 `docs/product/mvp-scope.md` 定义，包含注册登录、场景选择、语音对话、AI 回复、纠错总结、练习记录和移动端响应式。
  - 用户明确反馈页面不像手机 App，并确认采用“移动 App Shell + 任务训练流”方向。
  - 技术栈 React + Vite + Tailwind + FastAPI 可支持该调整，不需要引入 Native App。
- 风险或缺口：
  - 历史 `docs/product/phase1-tasks.md` 任务状态仍为 pending，与已合入主线的交付事实不一致。
  - 手机端 App 化调整属于 UI/体验重构，必须防止混入未确认的新业务能力。
- 推荐方案：
  - 保留 Web 技术栈。
  - 采用手机端 App Shell、单列任务流、底部主操作、移动触控标准。
  - 不新增后端接口和数据模型。
- 备选方案：
  - 只做响应式修补，不做 App Shell；不推荐，因为不能解决“像网页端”的核心问题。
- 是否需要用户确认：已确认采用“方案 A：移动 App Shell + 任务训练流”。

## 4. 需求范围

### 必须做

- 用户注册、登录、登出、JWT 会话保持。
- 首页展示 5 个场景任务，点击进入对话。
- 用户能完成至少 10 轮回合制英语对话。
- 浏览器 STT 语音输入与 TTS 播放。
- AI 回复与场景相关。
- 对话结束后展示纠错和课后总结。
- 首页展示练习记录，支持历史回放。
- 手机端优先 UI：375px-1440px 可用，无横向滚动，关键触控目标不小于 44px。
- 页面状态覆盖 Default / Loading / Empty / Error。

### 可选做

- 中式英语画像、训练任务标签、成长反馈等前端展示辅助。
- 公网 FRP 临时访问与截图验收证据。

### 暂不做

- Native App / 小程序。
- 多语种。
- 流式实时语音。
- 第三方登录。
- 独立历史管理页、社区、课程商城、教师端。
- 生产级 HTTPS、监控告警、正式发布流水线。

## 5. 业务规则

- 注册验证码 MVP 固定为 4 位数字，默认 `1234`。
- 密码必须 bcrypt 加密。
- 登录使用 JWT，会话需支持刷新保持。
- 场景为 5 个预设种子场景：面试、点餐、会议、旅行、日常。
- 对话消息需持久化，历史回放为只读模式。
- 总结与纠错需基于对话生成并持久化。
- DeepSeek API key 不得写入仓库，必须通过安全配置注入。
- 本地 SQLite `talkmate.db` 属运行数据，不作为代码交付物提交。

## 6. 验收口径

- 用户能完成完整流程：注册登录 -> 选择场景 -> 对话不少于 10 轮 -> 查看纠错 -> 查看总结 -> 回看历史。
- 登录、注册、首页、对话、总结页面在 375/390/430/768/1440 视口下无横向滚动和明显遮挡。
- 关键按钮触控目标不小于 44px。
- 前端构建通过。
- 后端模块测试通过。
- Playwright E2E 覆盖完整旅程、页面状态、多视口布局。
- P0/P1 缺陷关闭或有明确复测结论。

## 7. 风险与约束

| 风险 | 影响 | 应对 |
|---|---|---|
| 浏览器 Web Speech API 在 HTTP 或部分浏览器中受限 | 语音能力可能不可用 | 保留文字输入降级，生产环境建议 HTTPS |
| DeepSeek API 超时或 key 缺失 | AI 回复和总结生成受影响 | 后端超时降级，key 安全注入 |
| 历史任务状态文档未同步 | 接手人员误判阶段 | 以本文件、交付报告、Git 主线和测试报告为准 |
| App 化 UI 导航与图标依赖 | 已确认接受当前 UI 方向，但需保证依赖和构建同步 | `lucide-react` 已补入依赖清单，后续提交需纳入 UI 代码和依赖文件 |
