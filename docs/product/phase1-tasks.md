## 任务详情

### T-001：项目结构搭建 + 用户注册登录

**任务目标**：搭建前后端项目骨架，实现用户注册、登录、登出。

**范围边界**：
- 包含：前后端目录结构、JWT 认证、注册/登录/登出接口、登录页/注册页、.env 配置
- 不包含：第三方登录、密码找回、用户资料编辑

**允许修改目录**：`frontend/src/`（auth 相关）、`backend/app/`（auth/core/db）、`docs/api/`

**验收标准**：
- [ ] 用户能用账号密码注册（含简单验证码）
- [ ] 用户能用账号密码登录，返回 JWT
- [ ] 登录后刷新页面保持登录状态
- [ ] 点击登出清除会话
- [ ] 密码使用 bcrypt 加密
- [ ] 新增 docs/api/auth.md 接口文档

**测试重点**：注册重复用户名处理、错误密码提示、JWT 过期处理

**依赖与风险**：密码必须 bcrypt 加密，JWT 密钥通过 .env 配置

---

### T-002：场景选择页面（首页）

**任务目标**：实现首页，展示 5 个场景卡片，用户可选择场景进入对话。

**范围边界**：
- 包含：首页布局、5 个场景卡片、种子数据初始化、导航栏（Logo + 用户头像 + 退出）
- 不包含：自定义场景、场景编辑

**允许修改目录**：`frontend/src/pages/HomePage.tsx`、`backend/app/modules/scenario/`

**验收标准**：
- [ ] 首页展示 5 个场景卡片（面试/点餐/会议/旅行/日常）
- [ ] 点击场景卡片创建对话并进入对话页
- [ ] 种子数据通过数据库初始化写入
- [ ] 导航栏显示用户头像和退出按钮

**测试重点**：场景卡片布局、空态展示、点击跳转

---

### T-003：对话页面（文字模式）

**任务目标**：实现对话页面，支持文字输入对话。

**范围边界**：
- 包含：消息列表、文字输入框、发送按钮、对话创建接口、消息发送接口
- 不包含：语音输入、语音播放

**允许修改目录**：`frontend/src/pages/ConversationPage.tsx`、`backend/app/modules/conversation/`

**验收标准**：
- [ ] 从场景页进入对话页面
- [ ] 能输入文字并发送消息
- [ ] AI 回复后正确展示消息气泡（用户右侧、AI 左侧）
- [ ] 刷新后对话记录不丢失

**测试重点**：消息发送/接收、滚动行为、长文本展示

---

### T-004：AI 对话集成

**任务目标**：接入 DeepSeek API，实现场景化 AI 回复。

**范围边界**：
- 包含：DeepSeek API 调用封装、5 个场景的 prompt 模板、消息发送后调用 AI
- 不包含：流式回复、多轮上下文优化

**允许修改目录**：`backend/app/modules/ai-service/`

**验收标准**：
- [ ] 发送消息后 AI 能在 2 秒内回复
- [ ] AI 回复与当前场景相关
- [ ] AI 回复语言自然流畅
- [ ] API 超时有降级处理

**测试重点**：不同场景 prompt 效果、API 超时处理、错误降级

---

### T-005：语音交互（STT + TTS）

**任务目标**：接入浏览器 Web Speech API，实现语音输入和语音播放。

**范围边界**：
- 包含：语音识别（SpeechRecognition）、语音合成（SpeechSynthesis）、录音按钮 UI、波形动画
- 不包含：流式语音、外部 STT/TTS API

**允许修改目录**：`frontend/src/components/VoiceRecorder.tsx`、`frontend/src/components/VoicePlayer.tsx`、`frontend/src/utils/`

**验收标准**：
- [ ] 点击录音按钮能识别语音并转为文字
- [ ] AI 回复文字能自动播放语音
- [ ] 说话后 1 秒内显示文字

**测试重点**：浏览器兼容性（Chrome/Safari）、录音权限提示

---

### T-006：详细纠错 + 课后总结

**任务目标**：对话结束后生成纠错反馈和课后总结。

**范围边界**：
- 包含：AI 生成纠错（含语法解释）、AI 生成课后总结（含评分、词汇分析、改进建议）、总结展示页面
- 不包含：历史对比、进步曲线

**允许修改目录**：`frontend/src/pages/SummaryPage.tsx`、`backend/app/modules/conversation/`

**验收标准**：
- [ ] 对话结束后展示纠错反馈（至少 1 条，含语法解释）
- [ ] 展示课后总结（综合评分、语法问题、词汇分析、改进建议）
- [ ] 总结数据持久化到数据库

**测试重点**：总结生成逻辑、总结页面布局

---

### T-007：练习记录列表 + 对话回溯

**任务目标**：首页展示历史练习记录，支持回溯查看对话详情。

**范围边界**：
- 包含：练习记录列表（场景名、日期、评分）、点击记录查看对话和总结
- 不包含：筛选/排序、数据导出

**允许修改目录**：`frontend/src/pages/HomePage.tsx`、`backend/app/modules/conversation/`

**验收标准**：
- [ ] 首页能查看历史练习记录列表
- [ ] 点击记录能回溯查看完整对话
- [ ] 列表展示场景名、日期、评分

**测试重点**：空态展示、分页/加载更多

---

### T-008：响应式 UI 适配 + 4 种状态补齐

**任务目标**：所有页面适配移动端和 PC 端，补齐每页 4 种交互状态。

**范围边界**：
- 包含：所有已有页面的响应式布局调整（375px-1440px）、每页 Loading/Empty/Error 状态
- 不包含：深色模式、主题切换

**允许修改目录**：`frontend/src/`（样式和状态相关）

**验收标准**：
- [ ] 375px（手机）、768px（平板）、1440px（PC）均可正常使用
- [ ] 核心交互在手机端可单手操作
- [ ] 每页 4 种状态均已实现（Default / Loading / Empty / Error）
- [ ] 无水平滚动条、无文字溢出

**测试重点**：多视口测试、触摸交互
