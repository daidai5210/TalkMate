## 2. 项目背景

### 为什么要做

中国英语学习者超 3 亿，口语是最大的短板。线下外教贵（100-300元/小时）、线上真人平台有社交压力、现有 AI 产品对话机械且纠错不准。

竞品 Speak（speak.com）已验证市场——App Store 4.8 分、月活百万级，说明 AI 口语陪练需求真实且强烈。

### 目标用户

- **画像 A：大学生**（18-25 岁），CET-4/6 已过但口语差，需要面试模拟、日常对话练习
- **画像 B：职场新人**（22-30 岁），工作需要用英语但开口困难，需要会议/面试前快速提升信心

### 核心痛点

- **缺陪练**：没人一起练，线下贵，线上平台有社交压力
- **AI 体验差**：现有 AI 产品对话机械、纠错不准、缺乏系统性反馈
- **无法量化进步**：不知道自己口语水平，缺乏可见的能力提升曲线
- **缺少针对性场景**：面试、点餐、会议等场景化练习不足

*引用：详见 [业务调研报告](./docs/product/business-research.md)*

## 3. 项目目标

### 一句话目标

> 让用户能在指定场景下，与 AI 进行自然的英语口语对话练习，获得即时纠错和课后总结。

### 成功标准

- [ ] 用户能选择场景（面试/点餐/会议等）并完成一段完整对话
- [ ] AI 能实时回复用户语音输入，对话回合至少 10 轮不断
- [ ] 对话结束后能生成纠错反馈（语法/表达错误）
- [ ] 对话结束后能生成课后总结
- [ ] 端到端延迟（语音→AI回复）≤ 3 秒（回合制）

---

## 4. MVP 范围摘要

### 必须包含

- **场景选择**：提供 3-5 个预设场景（面试、点餐、会议、日常、旅行）
- **回合制语音对话**：用户说 → AI 回复的完整循环，浏览器内置 STT/TTS
- **语法/表达纠错**：AI 对话后即时指出错误并给出正确表达
- **课后总结**：每次练习生成结构化总结（表现评分 + 改进建议）
- **移动端优先响应式 UI**：手机为主，自适应 PC
- **用户基础信息**：登录/注册 + 练习记录

### 明确不做

- **多语种支持**（MVP 仅英语）
- **流式实时语音**（MVP 回合制即可）
- **社区/社交功能**
- **课程商城**
- **教师端/管理后台**
- **Native App / 小程序**
- **第三方登录**（微信/Google 等）

## 5. 约束条件

| 类型 | 约束 |
|------|------|
| 前端 | Web 端，移动端优先响应式设计，一套 UI 适配手机+PC |
| 后端 | 使用 DeepSeek API（deepseek-v4-flash），OpenAI 兼容协议 |
| STT/TTS | MVP 阶段使用浏览器内置 Web Speech API（SpeechRecognition + SpeechSynthesis），免费零成本 |
| 数据 | SQLite（MVP 阶段） |
| 部署 | 开发/测试在全栈主机（172.17.0.5），最终部署在 QA 主机（172.17.0.6） |
| 时间 | MVP 预计 2-3 个 Sprint |
| Git | 仅 SSH 推送，Token 已撤销 |
| 提交规范 | 所有 commit/issue/PR 使用中文，全周期持续交付，严禁突击提交 |

## 8. 风险与假设

| 风险 / 假设 | 影响 | 应对 |
|-------------|------|------|
| 浏览器 STT 对口音识别不准 | 高 | MVP 先用浏览器 API 验证，不行再升级外部 API |
| DeepSeek API 响应延迟高 | 中 | prompt 优化 + 控制上下文长度，超时降级处理 |
| 语音功能在部分浏览器不支持 | 中 | 提供文字对话降级模式（fallback） |
| 移动端 UI 适配复杂度高 | 中 | 使用 Tailwind 等响应式框架，全栈评估 |
