## 2. 项目背景

### 为什么要做

中国英语学习者超 3 亿，口语是最大短板。线下外教贵（100-300 元/小时）、线上真人平台有社交压力、现有 AI 产品对话机械且纠错不准。

目标用户：
- **大学生**（18-25 岁）：备考口语面试，不敢开口
- **职场人士**（22-30 岁）：工作需要英语，会议/面试前需快速提升
- **出国旅游人群**：旅行前口语突击，怕在国外沟通障碍

*引用：详见 [业务调研报告](./docs/product/business-research.md)*

### 核心痛点

- 缺陪练：没人一起练，线下贵，线上有社交压力
- AI 体验差：对话机械、纠错不准、缺乏系统性反馈
- 无法量化进步：不知道口语水平，缺乏可见的能力提升曲线
- 缺少针对性场景：面试/点餐/会议/旅行等场景化练习不足

---

## 3. 项目目标

### 一句话目标

> 让用户能在指定场景下，与 AI 进行语音口语对话练习，获得详细纠错和课后总结。

### 成功标准

- [ ] 用户能完成注册登录，进入场景选择
- [ ] 用户能选择场景并完成语音对话（≥10 轮）
- [ ] 对话结束后能查看详细纠错反馈（含语法解释）
- [ ] 对话结束后能查看详细课后总结（含评分、词汇分析、改进建议）
- [ ] 端到端延迟（语音→AI 回复播放）≤ 3 秒

---

## 4. MVP 范围摘要

### 必须包含

- **用户注册登录**：完整注册（用户名+密码+简单验证码），登录登出，JWT 会话
- **5 个场景全覆盖**：面试、点餐、会议、旅行、日常，场景卡片选择
- **回合制语音对话**：语音输入（STT）→ AI 回复 → 语音播放（TTS），≥10 轮
- **详细纠错反馈**：错误原文 → 正确表达 → 语法规则解释 → 为什么错 → 怎么改进
- **详细课后总结**：综合评分 + 语法问题 + 词汇分析 + 改进建议 + 下次练习重点
- **练习记录列表**：首页展示历史记录（场景、日期、评分），可回溯
- **移动端优先响应式**：375px-1440px 一套 UI

### 明确不做

- 多语种支持（MVP 仅英语）
- 流式实时语音（回合制）
- 个人信息页 / 设置页
- 独立对话记录管理页
- 社区/社交功能
- 课程商城
- 教师端/管理后台
- 第三方登录
- Native App / 小程序

## 5. 约束条件

| 类型 | 约束 |
|------|------|
| 前端 | React 18 + TypeScript + Vite + Tailwind CSS，移动端优先响应式 |
| 后端 | Python FastAPI + SQLAlchemy 2.0 + SQLite（MVP 阶段） |
| AI 集成 | DeepSeek v4-flash（OpenAI 兼容协议），openai Python SDK |
| STT/TTS | 浏览器 Web Speech API（SpeechRecognition + SpeechSynthesis），免费零成本 |
| 认证 | JWT（python-jose），密码 bcrypt 加密 |
| 部署 | 开发/测试在全栈主机（172.17.0.5），最终部署在 QA 主机（172.17.0.6） |
| 设计系统 | Indigo 品牌色，浅/深两套主题，所有颜色使用 tokens |
| Git | 仅 SSH 推送，Token 已撤销 |
| 提交规范 | 所有 commit/issue/PR 使用中文，全周期持续交付，严禁突击提交 |

## 8. 风险与假设

| 风险 / 假设 | 影响 | 应对 |
|-------------|------|------|
| 浏览器 STT 对口音识别不准 | 高 | MVP 先用浏览器 API 验证，不行再升级外部 STT API |
| DeepSeek API 响应延迟高 | 中 | prompt 优化 + 控制上下文长度，超时降级处理 |
| 语音功能在部分浏览器不支持 | 中 | 提供文字输入降级方案（fallback） |
| AI 纠错不够精准 | 中 | 通过 prompt 工程优化，后续可微调 |
| MVP 阶段 SQLite 性能 | 低 | MVP 用户量小足够，后续迁移 PostgreSQL |
